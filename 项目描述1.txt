我构建了 **EmbeddingForge** 这一套基于 Claude Code 的多智能体向量嵌入优化流水线，作为独立开发者同时维护多个语义检索系统（电商商品搜索、代码语义搜索、文档智能问答），每个系统的检索质量直接取决于向量嵌入模型的效果，传统嵌入优化依赖算法工程师手动选择预训练模型、设计微调策略、评估检索效果，整个过程需要深厚NLP知识且单次优化耗时3-5天，跨领域迁移效果差且难以量化改进收益。EmbeddingForge 通过 Analyzer → Trainer → Indexer → Evaluator 四个 AI Agent 实现嵌入分析→模型微调→索引构建→效果评估的全自动化闭环，每个 Agent 需要在完整的语料库上下文中结合向量检索原理进行长链推理，深入理解对比学习、难负例挖掘、HNSW索引、检索评估指标等向量检索核心技术，单次流水线运行消耗约 600-1000万Token，日常开发中我每天至少运行 3-5 次（覆盖新领域嵌入适配、检索效果调优、索引性能优化等场景），日均消耗超过 1500万 Token，月均消耗约 4.5亿 Token。该工具将嵌入优化周期从 3-5 天压缩到 25 分钟，Recall@10提升 35%，检索延迟降低 40%。技术栈基于 Python + Claude Code + DeepSeek API + FAISS，相关代码已全部开源至 GitHub（github.com/dongjieliang8-blip/embeddingforge），附有优化流程截图、检索效果对比和综合评分截图作为申请证明材料。
